What This Project
Demonstrates

⚡
Auto Scaling
Horizontal Pod Autoscaler dynamically adjusts replicas based on CPU and memory metrics in real-time.
🔄
Zero Downtime Deploys
Rolling updates ensure continuous availability with configurable surge and unavailability thresholds.
🛡️
Self Healing
Liveness and readiness probes automatically restart failed containers, maintaining declared desired state.
📦
Service Discovery
CoreDNS-powered internal service mesh for seamless pod-to-pod communication across namespaces.
🔒
RBAC Security
Role-based access control with least-privilege principles, Secrets encryption, and network policies.
📊
Observability
Integrated logging, metrics collection, and health dashboards for full cluster visibility.

Hands-on Commands

kubectl in Action

k8s-cluster — bash
# Apply the deployment manifest
$ kubectl apply -f deployment.yaml
deployment.apps/my-app created
service/my-app-svc created

# Verify pods are running
$ kubectl get pods -n production
NAME READY STATUS RESTARTS AGE
my-app-7d9fb4c89-x2kqp 1/1 Running 0 2m
my-app-7d9fb4c89-p8rvn 1/1 Running 0 2m
my-app-7d9fb4c89-tz4ms 1/1 Running 0 2m

# Scale the deployment
$ kubectl scale deployment my-app --replicas=5
deployment.apps/my-app scaled

# Check cluster nodes
$ kubectl get nodes
NAME STATUS ROLES AGE VERSION
